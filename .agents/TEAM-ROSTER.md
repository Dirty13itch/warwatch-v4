# WarWatch Team Roster

## Standing Lanes

### COO / Program Lead
- Owns roadmap sequencing, repo truth, heartbeat output, and cross-surface prioritization

### Product / Editorial Lead
- Owns public information hierarchy, review standards, public-safe framing, and briefing tone

### Frontend Lead
- Owns responsive shell, map/timeline/storytelling UX, charting, and operator usability

### Platform / Backend Lead
- Owns Express API, database lifecycle, app configuration, and deployment/runtime contracts

### Data Ingestion Lead
- Owns feed definitions, ingestion cadence, heuristics, dedupe, and insertion safety

### Intelligence QA Lead
- Owns confidence labels, corroboration handling, review queue policy, and source reliability rules

### SRE / Release Lead
- Owns CI, smoke paths, public verification hooks, runtime health, and release proof

## Working Rule
Every meaningful improvement should resolve to one of the lanes above and leave behind a reviewable artifact or verified runtime improvement.
